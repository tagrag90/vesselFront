# 블로그 플랫폼 심플 디자인 컨셉
**Studio_bada Vessel Design System v1.0**

---

## 🎨 디자인 철학

**"Minimalism with Purpose"**
- 불필요한 장식 제거, 콘텐츠에 집중
- 넓은 여백과 타이포그래피로 가독성 극대화
- 검은색/흰색 기반 단색 팔레트로 일관성 유지
- 미니멀한 인터랙션으로 사용자 경험 단순화

---

## 🎨 1. 색상 시스템

### 기본 팔레트
```
Primary (텍스트/배경)
- Black: #000000 (라이트 모드 텍스트, 다크 모드 배경)
- White: #FFFFFF (라이트 모드 배경, 다크 모드 텍스트)

Secondary (구조/구분)
- Grey: #F3F3F3 (라이트 모드) / #2A2A2A (다크 모드)
- Dark Grey: #6B6B6B (라이트 모드) / #E7E7E7 (다크 모드)

Accent (인터랙션)
- Purple: #8B46FF (라이트 모드) / #582C8E (다크 모드)
- Red: #FF4E4E (라이트 모드) / #991F1F (다크 모드) - 좋아요/강조용
```

### 사용 원칙
- **텍스트**: Black/White만 사용, Dark Grey는 보조 텍스트
- **배경**: White/Grey만 사용, 레이어 구분에만 Grey 활용
- **액센트**: Purple은 링크/호버, Red는 좋아요/알림에만 사용
- **보더**: Black만 사용, 두께 1px 고정

---

## 📝 2. 타이포그래피

### 폰트 계층
```
1. 본문: Pretendard (14px base, 16px line-height)
   - 가독성 최우선, 한국어/영문 모두 최적화
   
2. 제목: Pretendard Bold (18px ~ 50px)
   - 계층적 크기: h1(38px) > h2(28px) > h3(20px) > body(14px)
   
3. 액센트: HakgyoansimByeolbichhaneul (특별 섹션용)
   - Footer, About 등 특별한 섹션에만 제한적 사용
```

### 타이포그래피 규칙
- **행간**: 1.6 (본문), 1.4 (제목)
- **자간**: -0.01em (제목), 0 (본문)
- **폰트 크기**: 12px, 14px, 16px, 20px, 28px, 38px, 50px만 사용
- **두께**: 400 (본문), 500 (서브텍스트), 700 (제목)

## 📐 3. 레이아웃 시스템

### 그리드 구조
```
컨테이너 최대 너비: 900px (블로그 콘텐츠), 1280px (목록)
여백: 5vw (모바일) → 7vw (태블릿) → 10vw (데스크톱)
간격: 4px 단위 (4, 8, 12, 16, 24, 32, 48px)
```

### 블로그 카드 구조
```
모바일 (1열)
- 이미지: 16:9 비율, 전체 너비
- 콘텐츠: padding 20px, 여백 16px

태블릿 (2열)
- 간격: 24px
- 카드 너비: (100% - 24px) / 2

데스크톱 (3열)
- 간격: 24px
- 카드 너비: (100% - 48px) / 3
```

## 🎯 4. 컴포넌트 디자인 원칙

### 블로그 카드
```
구조:
- 이미지 (16:9) → 태그 → 제목 → 설명(선택) → 작성자 정보

스타일:
- 배경: White
- 보더: 1px solid Black
- 둥근 모서리: 8px
- 호버: 보더 색상 변경 (Purple) + 미세한 그림자

크기:
- 최소 높이: 없음 (콘텐츠 기반)
- 패딩: 20px
```

### 버튼
```
Primary (작성, 제출)
- 배경: Black
- 텍스트: White
- 둥근 모서리: 999px (full)
- 패딩: 12px 24px
- 호버: opacity 0.8

Secondary (취소, 보조)
- 배경: Grey
- 텍스트: Black
- 호버: Grey → Black (배경)

Text (링크)
- 배경: 없음
- 텍스트: Black
- 호버: Purple
- 언더라인: 호버 시만
```

### 입력 필드
```
기본:
- 배경: Grey
- 보더: 1px solid Grey
- 둥근 모서리: 8px
- 패딩: 16px 16px 16px 48px (아이콘 공간)

포커스:
- 배경: White
- 보더: 1px solid Black
- 아웃라인: 없음
```

## ✨ 5. 인터랙션 디자인

### 애니메이션
```
전환 효과:
- 기본: 100ms (색상, 배경)
- 페이지 전환: 300ms (fade)
- 호버: 150ms

애니메이션 원칙:
- 불필요한 애니메이션 제거
- 사용자 액션에만 반응
- 자연스러운 easing (ease-in-out)
```

### 인터랙티브 요소
```
호버:
- 버튼: 배경색/텍스트색 변경
- 링크: Purple + 언더라인
- 카드: 보더 Purple + 그림자

클릭:
- 피드백: 100ms opacity 변화
- 로딩: 스피너 표시
```

태블릿:
- 그리드: 2열
- 여백: 7vw

데스크톱:

## 🎭 7. 아이콘 사용 원칙

**Studio_bada 규칙 준수: 아이콘 사용 최소화**

### 허용 범위
```
필수 인터랙션만:
- 좋아요 (하트)
- 댓글 (말풍선)
- 공유 (링크)
- 검색 (돋보기)
- 알림 (벨)

제한:
- 장식용 아이콘 금지
- Flaticon 사용 최소화
- 가능한 텍스트/이모지로 대체
```

### 아이콘 스타일
```
크기: 16px (작은), 20px (기본), 24px (큰)
색상: Black/White/Dark Grey만 사용
스타일: Regular 또는 Rounded만 사용
```

## 📄 8. 블로그 페이지 특화 디자인

```

- 배너 이미지: 16:9, 전체 너비 (모바일), 둥근 모서리 (데스크톱)

- 메타 정보: 작성자 + 날짜, 간격 16px
- 본문: 900px 최대 너비, 행간 1.8, 폰트 18px
- 인터랙션 바: 상단/하단 구분선, 아이콘 버튼

여백:
- 상단: 40px
- 본문 간: 32px
- 하단: 48px

### 블로그 목록 페이지
```
레이아웃:
- 섹션 타이틀: 28px, 하단 여백 8px
- 설명: 14px, Dark Grey, 하단 여백 24px
- 그리드: 3열 (데스크톱), 간격 24px
- 페이지네이션: 중앙 정렬, 간격 16px

## 🎨 9. 다크 모드

### 컬러 변환
```
라이트 → 다크:
- Black ↔ White (텍스트/배경 교환)
- Grey: 밝은 회색 → 어두운 회색
- 액센트 색상: 약간 더 어둡게 조정
```

### 가독성 보장
```
대비:
- 텍스트/배경: 최소 7:1 (WCAG AAA)
- 보조 텍스트: 최소 4.5:1 (WCAG AA)

이미지:
- 다크 모드에서 약간 밝게 조정 (CSS filter)
```

## ✅ 10. 적용 우선순위

### Phase 1: 기본 구조 (최우선)
1. 색상 시스템 통일 (인라인 스타일 → Tailwind 클래스)
2. 타이포그래피 계층 정리 (폰트 크기 통일)
3. 레이아웃 여백/간격 표준화

### Phase 2: 컴포넌트 정리
4. 블로그 카드 디자인 통일
5. 버튼/입력 필드 스타일 통일
6. 인터랙션 애니메이션 최적화

### Phase 3: 세부 최적화
7. 아이콘 사용 최소화
8. 반응형 레이아웃 개선
9. 다크 모드 색상 조정
10. 접근성 개선 (키보드 네비게이션, ARIA)
